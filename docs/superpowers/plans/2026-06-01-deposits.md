# Deposits (Partial Payment) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let guests pay an agency-configured percentage deposit online now (via Kashier) and settle the balance on arrival, with the balance recorded and an admin "mark balance paid" action.

**Architecture:** Add a `payment_status` money axis (plus money/audit columns) to `bookings`, separate from the existing lifecycle `status`. Deposit math and USD→EGP conversion are computed server-side in pure, unit-tested helpers; the actual EGP charge and the FX rate used are recorded for audit. Guests choose deposit vs full at checkout (online only). Refunds are policy-text only in v1.

**Tech Stack:** Next.js 15 (App Router, server actions), TypeScript strict, Supabase (Postgres + service-role client), Kashier (EGP HPP), Vitest (added here for pure-logic tests).

**Spec:** `docs/superpowers/specs/2026-06-01-deposits-design.md`

**Testing philosophy:** This repo has no test runner today and has relied on `tsc --noEmit` + manual verification. Because this feature touches money, we add a lightweight Vitest setup and **TDD the pure logic only** (deposit math, FX rounding) — that is where mistakes cost real money. Server actions / webhook / UI are verified by `npm run typecheck` plus the manual checks noted in each task (mocking Supabase/Next internals would be brittle and low-value).

**Money invariants (hold across the whole feature):**
- All amounts at rest are **USD**. Only `charged_amount` / `charged_currency` / `fx_rate_used` describe the real EGP transaction.
- `amount_paid + balance_due == total_price` for every confirmed booking.
- The server NEVER trusts a client-supplied deposit percent or amount — it re-reads the percent from agency settings and recomputes.
- Reconciliation is computed from stored absolutes (not increments) so webhook retries are idempotent.

/**
 * Pure deposit math. No I/O. All amounts are USD (the app's at-rest currency).
 * The server is the only place allowed to compute these — the client never
 * supplies the percent or the split.
 */

export type PaymentChoice = 'deposit' | 'full';

export function round2(n: number): number {
  return Math.round((n + Number.EPSILON) * 100) / 100;
}

/** Clamp to an integer percent in [1, 100]; non-finite -> 1. */
export function clampPercent(p: number): number {
  if (!Number.isFinite(p)) return 1;
  return Math.min(100, Math.max(1, Math.floor(p)));
}

export interface DepositBreakdown {
  /** USD to charge online now. */
  depositUsd: number;
  /** USD still owed, collected on arrival. */
  balanceUsd: number;
  /** The percent actually applied, or null for a full payment. */
  depositPercent: number | null;
}

export function computeDepositBreakdown(input: {
  totalUsd: number;
  choice: PaymentChoice;
  percent: number;
}): DepositBreakdown {
  const total = round2(Math.max(0, input.totalUsd));

  if (input.choice !== 'deposit') {
    return { depositUsd: total, balanceUsd: 0, depositPercent: null };
  }

  const pct = clampPercent(input.percent);
  const depositUsd = round2((total * pct) / 100);
  const balanceUsd = round2(total - depositUsd);
  return { depositUsd, balanceUsd, depositPercent: pct };
}
```

/**
 * USD -> EGP foreign-exchange helper, shared by every server-side charge path
 * (checkout deposit, pay-via-link). Display-time conversion lives in the
 * client `use-currency` hook; this module is the authority for REAL charges.
 */

export const FALLBACK_USD_TO_EGP = 47.5;

const CURRENCY_API =
  'https://cdn.jsdelivr.net/npm/@fawazahmed0/currency-api@latest/v1/currencies/usd.json';

/** Live USD->EGP rate from the public CDN, falling back to a constant. */
export async function fetchUsdToEgp(): Promise<number> {
  try {
    const res = await fetch(CURRENCY_API, { next: { revalidate: 3600 } });
    if (!res.ok) return FALLBACK_USD_TO_EGP;
    const data = (await res.json()) as { usd?: Record<string, number> };
    const rate = data.usd?.egp;
    return typeof rate === 'number' && rate > 0 ? rate : FALLBACK_USD_TO_EGP;
  } catch {
    return FALLBACK_USD_TO_EGP;
  }
}

/** Convert a USD amount to an EGP charge, rounded to 2dp. Pure. */
export function usdToEgp(usd: number, rate: number): number {
  const safeRate = Number.isFinite(rate) && rate > 0 ? rate : FALLBACK_USD_TO_EGP;
  const safeUsd = Number.isFinite(usd) && usd > 0 ? usd : 0;
  return Math.round((safeUsd * safeRate + Number.EPSILON) * 100) / 100;
}
```

```ts
export async function applyVerifiedPaymentStatusChange(
  merchantOrderId: string,
  nextStatus: 'Confirmed' | 'Cancelled',
): Promise<void> {
  const supabase = createServiceRoleClient();

  if (nextStatus === 'Cancelled') {
    await supabase
      .from('bookings')
      .update({ status: 'Cancelled' })
      .eq('id', merchantOrderId);
    return;
  }

  // Positive payment: confirm + reconcile money from stored absolutes
  // (idempotent — safe to re-run on webhook retries).
  const { data } = await supabase
    .from('bookings')
    .select('total_price, balance_due, deposit_percent')
    .eq('id', merchantOrderId)
    .maybeSingle();

  const total = Number((data as { total_price?: number } | null)?.total_price ?? 0);
  const storedBalance = Number((data as { balance_due?: number } | null)?.balance_due ?? 0);
  const isDeposit =
    (data as { deposit_percent?: number | null } | null)?.deposit_percent != null;

  const paymentFields = isDeposit
    ? {
        payment_status: 'deposit_paid' as const,
        amount_paid: Math.round((total - storedBalance) * 100) / 100,
        balance_due: storedBalance,
      }
    : {
        payment_status: 'paid_in_full' as const,
        amount_paid: total,
        balance_due: 0,
      };

  await supabase
    .from('bookings')
    .update({ status: 'Confirmed', ...paymentFields })
    .eq('id', merchantOrderId);
}
```

Add to `src/lib/supabase/bookings.ts` (near other admin booking actions). It must be admin-gated the same way other admin mutations are — use the existing admin-client/membership helper this file already uses (e.g. `createAdminClient`); match the surrounding pattern.

/**
 * Admin action: record that a booking's outstanding balance has been
 * collected in person. Sets the booking paid-in-full.
 */
export async function markBalancePaid(bookingId: string): Promise<void> {
  const supabase = await createAdminClient(); // admin-membership gated, as elsewhere in this file
  const { data, error: readErr } = await supabase
    .from('bookings')
    .select('total_price')
    .eq('id', bookingId)
    .single();
  if (readErr || !data) throw new Error('Booking not found.');

  const total = Number((data as { total_price?: number }).total_price ?? 0);
  const { error } = await supabase
    .from('bookings')
    .update({
      payment_status: 'paid_in_full',
      amount_paid: total,
      balance_due: 0,
      balance_paid_at: new Date().toISOString(),
    })
    .eq('id', bookingId);
  if (error) throw new Error('Failed to mark balance paid.');
}
